## Executive Pulse — แดชบอร์ดการเข้าร่วมกิจกรรมของผู้บริหาร
ระบบบนเบราว์เซอร์สำหรับผู้บริหาร ออกแบบให้ใช้งานง่าย เปิดไฟล์ `executive-dashboard.html`
ได้ทันที (ไม่ต้องติดตั้ง) ข้อมูลฝังอยู่ในไฟล์ สรุปจากแบบสำรวจผู้บริหารเข้าร่วมกิจกรรมประจำสัปดาห์
(6 สัปดาห์ · 24 กิจกรรม · ผู้บริหาร 10 ท่าน) ครอบคลุม 4 มุมมอง:

- **ภาพรวมผู้บริหาร** — KPI การ์ด, แนวโน้มอัตราเข้าร่วมรายสัปดาห์, สัดส่วนสถานะ, อันดับรายบุคคล และไฮไลต์อัตโนมัติ
- **ตารางสถานะรวม** — เมทริกซ์ฮีตแมป ผู้บริหาร × กิจกรรม เห็นทุกอย่างในหน้าจอเดียว
- **รายสัปดาห์** — เลือกสัปดาห์ ดูรายละเอียดกิจกรรม วัน–เวลา–สถานที่ และผู้เข้าร่วม
- **รายบุคคล & อันดับ** — จัดอันดับการเข้าร่วม พร้อมประวัติรายกิจกรรมของผู้บริหารแต่ละท่าน
- **สร้างแบบสำรวจ** — ฟอร์มสร้างแบบสำรวจการเข้าร่วมกิจกรรมใหม่ตามโครงสร้างเดิม กด “สร้างฟอร์มอัตโนมัติ”
  เพื่อเติมรายชื่อผู้บริหารทั้ง 10 ท่านให้พร้อมติ๊ก เข้าร่วม/ไม่เข้าร่วม แล้วบันทึกเข้าแดชบอร์ด (เก็บใน
  เบราว์เซอร์ ผลรวมอัปเดตทุกหน้า), ส่งออก JSON/CSV หรือพิมพ์แบบฟอร์มสำหรับใช้งานจริง
  - **ลิงก์ให้ตอบ + กำหนดปิดรับ (deadline)** — สร้างลิงก์แชร์ (LINE/อีเมล) ผู้บริหารเปิดลิงก์ เลือกชื่อตนเอง
    และติ๊กการเข้าร่วม ระบบ**ปิดรับอัตโนมัติเมื่อถึงกำหนด** (แสดงหน้า “ปิดรับคำตอบแล้ว”) — ทำงานแบบ
    serverless: ข้อมูลแบบสำรวจฝังในลิงก์ (URL hash) ใช้ได้บน GitHub Pages ทันทีโดยไม่ต้องมีเซิร์ฟเวอร์
  - **นำเข้าคำตอบ** — ผู้ตอบกด “ส่งคำตอบ” ได้รหัสคำตอบส่งกลับ ผู้ดูแลวางรหัสเพื่อรวมผลเข้าแดชบอร์ด
  - **นำเข้า Excel/CSV** — อัปโหลดไฟล์ (รูปแบบเดียวกับที่ส่งออก) เพื่อเติมข้อมูลลงฟอร์มอัตโนมัติ

รองรับโหมดมืด, ปรับขนาดหน้าจอ (มือถือ/แท็บเล็ต/เดสก์ท็อป), พิมพ์/บันทึก PDF และส่งออก CSV
อัปเดตข้อมูลตั้งต้นได้โดยแก้ JSON ในแท็ก `<script id="data">` ภายในไฟล์ ส่วนแบบสำรวจที่สร้างใหม่
จะถูกเก็บไว้ในเบราว์เซอร์ (localStorage)

## DocIntel — Document Intelligence Platform
ต้นแบบ frontend ใช้งานได้ทันที (เปิด `doc-intelligence.html`) ครอบคลุม: Dashboard, คลังเอกสาร,
ค้นหาขั้นสูง, Document Viewer, เปรียบเทียบเอกสาร, AI Analyzer, AI Chat, Knowledge Graph, Report Center

### เชื่อม Backend จริง
ดูวิธี deploy และเชื่อมต่อ API ที่ [`backend/README.md`](backend/README.md)
- ฐานข้อมูล D1 `psru-docintel` (สร้าง + ใส่ข้อมูลตัวอย่างแล้ว)
- REST API: เอกสาร CRUD + ค้นหาเต็มข้อความภาษาไทย (FTS5 trigram)
- ตั้งค่า `API_BASE` ใน `doc-intelligence.html` ให้ชี้ไปยัง Worker เพื่อใช้ข้อมูลจริง
