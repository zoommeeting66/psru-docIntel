# 02 — User Journey

แสดงเส้นทางผู้ใช้แต่ละบทบาท ตั้งแต่ก่อนเกิดเหตุ → ระหว่างเหตุ → หลังเหตุ
พร้อมสถานการณ์จำลอง (scenarios) ทั้งแบบ Online และ Offline

---

## 2.1 Persona สรุป
| Persona | บริบท | Pain point | สิ่งที่คาดหวัง |
|---------|-------|------------|----------------|
| **สมชาย** พนักงานภาคสนาม | ทำงานพื้นที่อับสัญญาณ | กดขอความช่วยเหลือยาก, ไม่มีเน็ต | ปุ่มเดียวส่งได้แม้ไม่มีเน็ต |
| **หัวหน้าวันดี** | ดูแลทีม 12 คน | ไม่รู้ว่าลูกทีมมีเหตุ | แจ้งเตือนทันที + เห็นตำแหน่ง |
| **รปภ. ก้อง** | เวรกลางคืน | ได้ข้อมูลไม่ครบตอนเข้าเหตุ | เคสเด้งพร้อมแผนที่/ข้อมูลจำเป็น |
| **ศูนย์ควบคุม นิตยา** | คุมหลายเหตุพร้อมกัน | จัดลำดับยาก, ไม่มีภาพรวม | Dashboard + จัดลำดับ + dispatch |

---

## 2.2 Journey: บุคลากรทั่วไป (Employee)

### ก่อนเกิดเหตุ (Onboarding)
1. ติดตั้งแอป → Login ด้วยบัญชีองค์กร (SSO)
2. แอปดึงโปรไฟล์จาก HR อัตโนมัติ (ชื่อ, หน่วยงาน, หัวหน้า)
3. ให้ **Consent** ข้อมูลสุขภาพ + ผู้ติดต่อฉุกเฉิน (เลือกได้ว่าจะเปิดเผยอะไร)
4. ให้สิทธิ์ตำแหน่ง (Always/While Using), แจ้งเตือน, Bluetooth
5. ทดสอบปุ่ม SOS ในโหมดซ้อม (test mode) ไม่ส่งจริง

### ระหว่างเกิดเหตุ (Online)
```
กดค้างปุ่ม SOS 3 วิ ─▶ สั่น+เสียงยืนยัน ─▶ (เลือกประเภทเหตุ/ข้าม)
   │                                          │
   ▼                                          ▼
แอปแนบ: ตัวตน, หน่วยงาน, GPS, แบต, เวลา ─▶ ส่งเข้า Backend (HTTPS)
   │
   ▼
หน้าจอแสดง "ส่งสำเร็จ • ศูนย์รับเรื่องแล้ว" + ปุ่มยกเลิก (15 วิ) + live status
```
- มี **cancel window 15 วินาที** กันกดผิด (ยกเลิกได้ก่อนแจ้งเต็มรูปแบบ — แต่ศูนย์ยังเห็น log)
- หลังส่ง: ผู้ใช้เห็นสถานะ real-time (รับเรื่อง → กำลังไปช่วย → ถึงที่ → ปิดเหตุ)
- สามารถส่งข้อความ/อัปเดตอาการเพิ่มเติม, แชร์ตำแหน่งต่อเนื่อง (live location)

### ระหว่างเกิดเหตุ (Offline)
```
กดค้าง SOS ─▶ แอปตรวจ "ไม่มีเน็ต" ─▶ บันทึกลง Outbox (encrypted)
   │
   ├─▶ ลอง SMS ไปเบอร์ศูนย์ (ถ้ามีสัญญาณมือถือ)
   ├─▶ ลอง BLE Mesh กระจายหาเครื่องใกล้เคียงให้ช่วย relay
   ├─▶ ลอง Wi-Fi Direct หาอุปกรณ์ใกล้
   └─▶ เก็บไว้ Store & Forward → ส่งทันทีเมื่อกลับมีสัญญาณ
หน้าจอแสดงช่องทางที่ส่ง/กำลังส่ง + สถานะแต่ละช่องทางแบบโปร่งใส
```

### หลังเกิดเหตุ
- ได้รับการแจ้ง "ปิดเหตุแล้ว" + แบบสอบถามสั้น (optional)
- ดูประวัติการแจ้งเหตุของตนเองได้

## 2.3 Journey: หัวหน้างาน (Supervisor)
1. รับ Push/SMS "ลูกทีม [ชื่อ] แจ้งเหตุ [ประเภท] ที่ [ตำแหน่ง]"
2. เปิดแอป/เว็บ ดูรายละเอียด + ตำแหน่งบนแผนที่
3. กด "รับทราบ/กำลังประสาน" → สถานะอัปเดตให้ทุกฝ่ายเห็น
4. ประสานงานกับ รปภ./ศูนย์ ผ่านช่อง chat ของเคส
5. ติดตามจน ปิดเหตุ → เห็น After-action summary ของทีม

---

## 2.4 Journey: เจ้าหน้าที่ รปภ. (Responder)
1. ได้รับ Alert real-time (เสียง+Push) พร้อมการ์ดเคส
2. เห็นแผนที่ + เส้นทาง + อาคาร/ชั้น/ห้อง + ข้อมูลจำเป็น (กรุ๊ปเลือด/โรคที่ยินยอม)
3. กด **"รับเคส (Accept)"** → ระบบ assign + เริ่มจับเวลา response
4. อัปเดตสถานะภาคสนาม: `enroute → onsite → resolved`
5. บันทึกผลการช่วยเหลือ + แนบรูป/หมายเหตุ → ปิดเคส (หรือส่งต่อ escalate)

---

## 2.5 Journey: ศูนย์ควบคุม / แอดมิน (Control / Admin)
1. Dashboard แสดงเคสใหม่เด้งขึ้นบนสุด + เสียงเตือน + หมุดบนแผนที่
2. ตรวจสอบ/จัดลำดับความเร่งด่วน (severity) และ **Dispatch** ผู้รับผิดชอบ
3. มอนิเตอร์หลายเคสพร้อมกัน, ติดตาม response time
4. หากเหตุใหญ่ → ส่ง **Broadcast** เตือนบุคลากรในพื้นที่ (อพยพ/หลีกเลี่ยงโซน)
5. ปิดเหตุ → ระบบสร้างรายงานหลังเหตุการณ์ (after-action report) อัตโนมัติ

---

## 2.6 Cross-cutting Scenarios

### Scenario A — เจ็บป่วยฉุกเฉินในอาคาร (Online)
สมชายวูบที่ชั้น 4 → กดค้าง SOS → ระบบระบุ "อาคาร A ชั้น 4" จาก BLE beacon →
ศูนย์เห็นเคสใน 4 วิ พร้อมข้อมูล "แพ้ยาเพนนิซิลลิน (ยินยอมเปิดเผย)" → dispatch รปภ.+พยาบาล →
หัวหน้าและผู้ติดต่อฉุกเฉินได้รับแจ้ง → ปิดเหตุใน 12 นาที → auto report

### Scenario B — อุบัติเหตุพื้นที่อับสัญญาณ (Offline → Online)
สมชายประสบเหตุในพื้นที่ไม่มีเน็ต → SMS ส่งไม่ออก (ไม่มีสัญญาณมือถือ) →
BLE Mesh กระจายถึงเครื่องเพื่อนร่วมงานห่าง 60 ม. ที่มีเน็ต → relay เข้า Backend →
เคสถูกสร้าง พร้อม flag "via BLE relay, ตำแหน่งล่าสุดก่อนสัญญาณหาย" →
เมื่อสมชายกลับมามีสัญญาณ Outbox sync เพิ่มข้อมูล/ตำแหน่งล่าสุดเข้าเคสเดิม (dedupe)

### Scenario C — เหตุใหญ่หลายคน (Mass incident)
ไฟไหม้อาคาร B → หลายคนกด SOS → ระบบ cluster เคสตามพื้นที่/เวลา →
ศูนย์เห็นเป็น Incident group → Broadcast "อพยพอาคาร B" → ติดตามจำนวนคนที่ยัง active
